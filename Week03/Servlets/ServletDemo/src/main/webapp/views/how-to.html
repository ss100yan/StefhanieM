HTML Fundamentals

What is HTML?

HTML stands for Hypertext Markup Language. It is interpreted by the browser -- not compiled. It is NOT a programming language. In any case, it is used to create the structure of web pages. Each element you see on a web bpage is there becauase of HTML (and, yes, the formal name for these components of a web page is "element".

What is the basic structure of an HTML document?

An HTML document has a standard "skeleton" that consists of several opening and closing tags(e.g. "<>"). These tags include:

  1. A Doctype Declaration: This tells the browser what type of document to expect. Note that the Doctype Declaration is not specific to HTML.
  2. A Root Tag: The root tag of any HTML document is the "html" tag.
  3. A Head Tag: This tag contains metadata about the document. This metadata includes information about the set of characters being used and the page's title.
  4. A Body Tag: The body tag contains other tags which add the actual structure to the page. Here you'll find paragraph tags, div tags, and a wide variety of other commonly used tags.

How are HTML elements rendered?

Recall that HTML is an interpreted language. This means that your browser has an interpreter which knows how to make sense of it -- no compilation needed. Well, when your browser is interpreting HTML, it renders each element on the page sequentially. In other words, it steps through the HTML document and reads each element sequently (line by line). And this isn't just true of HTML elements -- it's also true of any scripts you might load.
